# LuxWeb Module: 3D & WebGL

> **Read when**: The project requires Three.js, React Three Fiber, 3D scenes, shaders, particles, or post-processing effects. If the project doesn't use 3D, skip this file entirely.

---

## When to Use 3D

- Agency/portfolio sites needing a "wow" factor
- Product showcase pages (rotating 3D product, exploded views)
- Hero sections where a static image isn't enough
- Interactive data visualizations
- Backgrounds that need to respond to mouse/scroll

**Do NOT use 3D for**: standard business websites, blogs, e-commerce product listings, dashboards, or anywhere it would slow load time without proportional value.

---

## Architecture: 3D + HTML Coexistence

The fundamental pattern: ONE persistent R3F `<Canvas>` lives behind ALL HTML content. It's `position: fixed` and fills the viewport. HTML sections scroll over it with `position: relative`.

```
┌────────────────────────────────────────┐
│  Canvas (fixed, z-0)                   │
│  ┌──────────────────────────────────┐  │
│  │  3D Scene (responds to scroll)   │  │
│  │  - particles, meshes, lights     │  │
│  │  - camera moves with scroll      │  │
│  └──────────────────────────────────┘  │
│                                        │
│  HTML (relative, z-1, scrollable)      │
│  ┌──────────────────────────────────┐  │
│  │  Hero Section (transparent bg)    │  │
│  │  About Section (semi-transparent) │  │
│  │  Work Section (transparent bg)    │  │
│  │  CTA Section (transparent bg)     │  │
│  │  Footer (opaque bg)              │  │
│  └──────────────────────────────────┘  │
└────────────────────────────────────────┘
```

### Persistent Canvas

```tsx
// components/canvas/persistent-canvas.tsx
"use client";

import { Canvas } from "@react-three/fiber";
import { Preload, AdaptiveDpr, AdaptiveEvents } from "@react-three/drei";
import { Suspense } from "react";
import { SceneController } from "./scenes/scene-controller";
import { PostProcessingStack } from "./effects/post-processing-stack";

export function PersistentCanvas() {
  return (
    <div className="fixed inset-0 z-0" aria-hidden="true">
      <Canvas
        gl={{
          antialias: true,
          alpha: true,
          powerPreference: "high-performance",
          stencil: false,
        }}
        camera={{ position: [0, 0, 5], fov: 45 }}
        dpr={[1, 2]}
        style={{ pointerEvents: "none" }}
      >
        <Suspense fallback={null}>
          <SceneController />
          <PostProcessingStack />
          <AdaptiveDpr pixelated />
          <AdaptiveEvents />
          <Preload all />
        </Suspense>
      </Canvas>
    </div>
  );
}
```

- `alpha: true` — canvas is transparent, HTML background shows through
- `pointerEvents: "none"` — clicks pass through to HTML. Use R3F's event system for 3D interaction.
- `aria-hidden="true"` — 3D is decorative. Screen readers ignore it.
- Dynamic import with `ssr: false`:

```tsx
const PersistentCanvas = dynamic(
  () => import("@/components/canvas/persistent-canvas").then(m => m.PersistentCanvas),
  { ssr: false }
);
```

### Zustand Store (3D State Bridge)

3D components read scroll/mouse state from Zustand every frame via `useFrame`. HTML components write to it.

```typescript
// store/use-app-store.ts
import { create } from "zustand";

interface AppState {
  scrollProgress: number;      // 0–1 normalized page scroll
  currentSection: number;      // Which section is active
  mousePosition: { x: number; y: number }; // -1 to 1 normalized
  isLoading: boolean;
  cursorVariant: "default" | "hover" | "action" | "hidden";

  setScrollProgress: (v: number) => void;
  setCurrentSection: (v: number) => void;
  setMousePosition: (x: number, y: number) => void;
  setIsLoading: (v: boolean) => void;
  setCursorVariant: (v: AppState["cursorVariant"]) => void;
}

export const useAppStore = create<AppState>((set) => ({
  scrollProgress: 0,
  currentSection: 0,
  mousePosition: { x: 0, y: 0 },
  isLoading: true,
  cursorVariant: "default",

  setScrollProgress: (v) => set({ scrollProgress: v }),
  setCurrentSection: (v) => set({ currentSection: v }),
  setMousePosition: (x, y) => set({ mousePosition: { x, y } }),
  setIsLoading: (v) => set({ isLoading: v }),
  setCursorVariant: (v) => set({ cursorVariant: v }),
}));
```

## 3D Objects

### Hero Geometry (Floating Mesh)

A geometric shape floating in the hero area:

- Slowly auto-rotates on Y axis.
- Tilts toward mouse position (lerped for smoothness).
- Custom shader material with animated color.
- Surrounded by small floating particles.

```typescript
// components/canvas/objects/hero-mesh.tsx
export function HeroMesh() {
  const meshRef = useRef<THREE.Mesh>(null!);
  const materialRef = useRef<THREE.ShaderMaterial>(null!);

  useFrame((state, delta) => {
    const { mousePosition } = useAppStore.getState();

    // Auto-rotate
    meshRef.current.rotation.y += delta * 0.15;

    // Mouse-reactive tilt
    meshRef.current.rotation.x = THREE.MathUtils.lerp(
      meshRef.current.rotation.x,
      mousePosition.y * 0.2,
      0.05
    );

    // Update shader time uniform
    materialRef.current.uniforms.uTime.value = state.clock.elapsedTime;
  });

  return (
    <mesh ref={meshRef}>
      <icosahedronGeometry args={[1.5, 4]} />
      <shaderMaterial ref={materialRef} /* ... */ />
    </mesh>
  );
}
```

Geometry options: `IcosahedronGeometry`, `TorusKnotGeometry`, `OctahedronGeometry`, or import a `.glb` model via `useGLTF`.

### Particle Field

An `InstancedMesh` with hundreds to thousands of small particles:

```typescript
export function ParticleField({ count = 1500 }) {
  const meshRef = useRef<THREE.InstancedMesh>(null!);
  const dummy = useMemo(() => new THREE.Object3D(), []);

  // Pre-compute random positions and velocities
  const particles = useMemo(() => {
    return Array.from({ length: count }, () => ({
      position: [
        (Math.random() - 0.5) * 20,
        (Math.random() - 0.5) * 20,
        (Math.random() - 0.5) * 20,
      ],
      speed: Math.random() * 0.5 + 0.1,
      offset: Math.random() * Math.PI * 2,
    }));
  }, [count]);

  useFrame((state) => {
    const { scrollProgress } = useAppStore.getState();
    const time = state.clock.elapsedTime;

    particles.forEach((p, i) => {
      // Drift with sine/cosine
      const x = p.position[0] + Math.sin(time * p.speed + p.offset) * 0.3;
      const y = p.position[1] + Math.cos(time * p.speed + p.offset) * 0.3;
      const z = p.position[2];

      // Spread outward with scroll
      const spread = 1 + scrollProgress * 0.5;

      dummy.position.set(x * spread, y * spread, z * spread);
      dummy.updateMatrix();
      meshRef.current.setMatrixAt(i, dummy.matrix);
    });

    meshRef.current.instanceMatrix.needsUpdate = true;
  });

  return (
    <instancedMesh ref={meshRef} args={[undefined, undefined, count]}>
      <sphereGeometry args={[0.015, 8, 8]} />
      <meshBasicMaterial color="#ffffff" transparent opacity={0.4} />
    </instancedMesh>
  );
}
```

### 3D Image Gallery

Textured planes in a circular arrangement:

```typescript
export function Gallery3D({ projects }: { projects: ProjectData[] }) {
  const groupRef = useRef<THREE.Group>(null!);
  const textures = useTexture(projects.map((p) => p.imagePath));

  useFrame(() => {
    const { scrollProgress } = useAppStore.getState();
    // Rotate gallery with scroll
    groupRef.current.rotation.y = scrollProgress * Math.PI * 2;
  });

  return (
    <group ref={groupRef}>
      {projects.map((project, i) => {
        const angle = (i / projects.length) * Math.PI * 2;
        const radius = 4;
        return (
          <mesh
            key={project.id}
            position={[Math.sin(angle) * radius, 0, Math.cos(angle) * radius]}
            rotation={[0, -angle + Math.PI, 0]}
          >
            <planeGeometry args={[2.5, 1.6]} />
            <meshStandardMaterial map={textures[i]} />
          </mesh>
        );
      })}
    </group>
  );
}
```

### Exploded Diagram

A `.glb` model that separates into parts on scroll:

```typescript
export function ExplodedModel({ modelPath }: { modelPath: string }) {
  const { scene } = useGLTF(modelPath);
  const parts = useRef<{ mesh: THREE.Mesh; homePos: THREE.Vector3; explodedPos: THREE.Vector3 }[]>([]);

  useEffect(() => {
    scene.traverse((child) => {
      if (child instanceof THREE.Mesh) {
        const home = child.position.clone();
        const direction = home.clone().normalize();
        const exploded = home.clone().add(direction.multiplyScalar(2));
        parts.current.push({ mesh: child, homePos: home, explodedPos: exploded });
      }
    });
  }, [scene]);

  useFrame(() => {
    const { scrollProgress } = useAppStore.getState();
    // Map scroll to 0–1 for this section only
    const explodeAmount = THREE.MathUtils.clamp((scrollProgress - 0.6) / 0.2, 0, 1);

    parts.current.forEach(({ mesh, homePos, explodedPos }) => {
      mesh.position.lerpVectors(homePos, explodedPos, explodeAmount);
    });
  });

  return <primitive object={scene} />;
}
```

### Particle Convergence (CTA Effect)

Particles scatter, then converge into a shape:

- Target positions sampled from a mesh (text geometry, logo, or shape).
- Current positions lerp toward targets based on scroll progress.
- Use `InstancedMesh` for performance.

## Shaders

### Vertex: Noise Displacement

```glsl
uniform float uTime;
uniform float uStrength;
uniform float uFrequency;

// Include snoise function (paste or use glsl-noise import)

void main() {
  float noise = snoise(position * uFrequency + uTime * 0.2);
  vec3 displaced = position + normal * noise * uStrength;
  gl_Position = projectionMatrix * modelViewMatrix * vec4(displaced, 1.0);
}
```

### Fragment: Gradient from Displacement

```glsl
uniform vec3 uColor1;
uniform vec3 uColor2;
varying float vDisplacement;

void main() {
  float mix = smoothstep(-0.5, 0.5, vDisplacement);
  vec3 color = mix(uColor1, uColor2, mix);
  gl_FragColor = vec4(color, 1.0);
}
```

### Fragment: Fresnel Edge Glow

```glsl
uniform vec3 uGlowColor;
uniform float uIntensity;
varying vec3 vNormal;
varying vec3 vViewDir;

void main() {
  float fresnel = pow(1.0 - abs(dot(normalize(vNormal), normalize(vViewDir))), 3.0);
  gl_FragColor = vec4(uGlowColor * fresnel * uIntensity, fresnel * 0.7);
}
```

## Post-Processing

```tsx
import { EffectComposer, Bloom, ChromaticAberration, Noise, Vignette } from "@react-three/postprocessing";

export function PostProcessingStack() {
  return (
    <EffectComposer>
      <Bloom intensity={0.4} luminanceThreshold={0.6} mipmapBlur />
      <ChromaticAberration offset={[0.0004, 0.0004]} />
      <Noise opacity={0.025} />
      <Vignette offset={0.1} darkness={0.7} />
    </EffectComposer>
  );
}
```

| Effect | Purpose | Performance Cost |
|--------|---------|-----------------|
| Bloom | Glow on emissive/bright areas | Medium |
| Chromatic Aberration | Analog lens feel at edges | Low |
| Film Grain (Noise) | Texture in dark areas | Low |
| Vignette | Darkens edges, focuses center | Low |

Max 4 effects active simultaneously. Disable ALL post-processing on mobile.

## Performance Rules (3D-Specific)

1. **Geometry budget**: Max 100K triangles visible at any time.
2. **InstancedMesh**: Use for anything with >10 identical objects (particles, repeated shapes).
3. **Pre-allocate**: Never create `new Vector3()`, `new Color()`, or `new Matrix4()` inside `useFrame`. Create once with `useMemo`, reuse.
4. **Dispose**: Always `dispose()` geometries, materials, and textures in cleanup. Memory leaks crash tabs.
5. **useFrame budget**: Keep ALL `useFrame` callbacks under 2ms combined per frame.
6. **Texture sizes**: Max 2048×2048 for hero, 1024×1024 for secondary. Compress to KTX2 or WebP.
7. **Preload models**: `useGLTF.preload("/models/model.glb")` at module level.
8. **Freeze when off-screen**: Use Intersection Observer to disable `useFrame` for sections not in viewport.
9. **GPU detection**: Use `useDetectGPU()` from drei to scale quality:

```typescript
const gpu = useDetectGPU();
const quality = gpu.tier >= 2 ? "high" : gpu.tier >= 1 ? "medium" : "low";
// Adjust: particle count, segment count, post-processing, texture resolution
```

### Mobile Fallbacks

- Reduce particles: 1500 → 300
- Disable post-processing entirely
- Lower geometry segments: 128×128 → 32×32
- Set `dpr={[1, 1.5]}` instead of `[1, 2]`
- Disable mouse-reactive 3D behaviors; use gentle auto-animation instead
- Disable custom cursor
- Consider replacing 3D scene with a static image or CSS animation on very low-end devices

## Critical Anti-Patterns (3D)

- ❌ NEVER render Canvas inside a section component. One persistent canvas, always.
- ❌ NEVER update Three.js objects from React state. Use refs + `useFrame`.
- ❌ NEVER allocate objects inside `useFrame`. Pre-allocate and reuse.
- ❌ NEVER skip `dispose()`. Memory leaks will crash the browser.
- ❌ NEVER use `position: absolute` for the canvas. Must be `position: fixed`.
- ❌ NEVER add pointer-events to the canvas div.
- ❌ NEVER let the 3D overpower the content. It's atmosphere, not the message.
- ❌ NEVER ship without testing on a mid-tier Android. 30fps minimum or reduce complexity.
- ❌ NEVER use Three.js default background. Always `alpha: true`.
